What it does

Flowmatic NC reads the program as an engineer would: motion, tool, feed, index angle, cumulative time and the relationship between code and geometry.

  • G00/G01/G02/G03 parsing and 3D path visualization
  • Cutting, rapid motion and tool-change CT breakdown
  • Tool, WCS and INDEX state tracking
  • STL/STEP-derived model overlay and motion review

NC demo media slot Add a screen recording, UI screenshot, before/after toolpath or customer case through the built-in media editor.

What it does

Flowmatic CT recognizes defined equipment states from video, timestamps transitions and automatically creates the event structure required for line balancing.

  • Camera/MP4 input and ROI-based event recognition
  • Automatic start, end and duration event log
  • Gantt chart, Worst-3 and cycle comparison
  • Repeatable CT evidence for improvement reviews
▶
CT recognition demo slot Add the recognition overlay, Gantt output, event log or a short line-balancing case video.

What it targets

The goal is not to overbuild a premium vision cell. It is to make repeatable inspection available where manual inspection is expensive, inconsistent or difficult to record.

  • Borescope and USB-camera multi-capture workflow
  • Alignment correction for camera and fixture variation
  • Burr, chip residue and appearance anomaly screening
  • Inspection evidence linked to production events
◎
Vision sample media slot Add normal/defect image pairs, borescope captures, detection overlays or a proof-of-concept video.

What it targets

Instead of assigning one fixed automation unit to every line, Flowmatic AMR treats mobile robots as a shared production resource and dispatches them according to line demand.

  • Input magazine replenishment and finished-part removal
  • Shared dispatch across multiple production lines
  • AMR + cobot task orchestration
  • Expandable interface for inventory and vision events
